# 🎮 테트리스 리뉴얼 개발 일지 - 2일차

기존에 제작했던 멀티플레이 테트리스 게임을 분석하고 리뉴얼 방향을 결정하는 시간을 가졌다.

기존 프로젝트는 기본적인 게임 기능은 정상적으로 동작하였지만, 사용자 경험과 유지보수성 측면에서 여러 개선 사항이 존재하였다.

이번 포스트에서는 현재 프로젝트의 문제점을 분석하고 향후 리뉴얼 계획을 정리하고자 한다.

---

## 🎯 포스팅 목표

* 기존 프로젝트 분석
* 문제점 파악
* 리뉴얼 목표 설정
* 개선 방향 수립

---

## 🔍 기존 프로젝트 분석

프로젝트를 점검한 결과 게임 자체는 정상적으로 동작하였지만 여러 부분에서 개선이 필요하다고 판단하였다.

특히 서버 구조, 사용자 인터페이스, 데이터베이스 설계 측면에서 보완이 필요하였다.

---

## ❌ 현재 프로젝트의 주요 문제점

### 1. 서버 구조의 복잡성

기존 프로젝트는 대부분의 서버 코드가 `server.js`에 집중되어 있었다.

```bash
server.js
```

사용자 처리, 데이터베이스 연결, 게임 로직 등이 하나의 파일에 작성되어 있어 코드 관리가 어려웠다.

#### 문제점

* 코드 가독성 저하
* 기능 추가 시 수정 범위 증가
* 버그 발생 시 원인 추적 어려움

#### 개선 방향

기능별 모듈 분리를 통해 유지보수성을 향상시킨다.

```bash
config/
└── db.js

routes/
├── user.js
└── stats.js
```

---

### 2. 오래된 사용자 인터페이스

기존 UI는 기능 구현에 집중되어 있어 시각적인 완성도가 부족하였다.

#### 문제점

* 단조로운 디자인
* 낮은 몰입감
* 게임의 경쟁력 부족

#### 개선 방향

* 로비 UI 전면 개편
* 대기방 UI 개선
* 게임 화면 리디자인
* 네온 스타일 적용

---

### 3. 데이터베이스 구조 개선 필요

기존 데이터베이스는 사용자 정보와 전적 정보가 하나의 테이블에 저장되어 있었다.

#### 문제점

* 데이터 중복 발생 가능
* 관리 효율 저하
* 기능 확장 어려움

#### 개선 방향

정규화를 적용하여 테이블을 분리한다.

```text
user
└─ 계정 정보

user_stats
└─ 전적 및 통계 정보
```

---

### 4. 게임 진행 흐름 개선 필요

게임 진행 과정에서 사용자 경험을 저해하는 요소들이 존재하였다.

#### 문제점

* 대기방 정보 표시 오류
* 화면 전환 과정의 불편함
* 게임 종료 후 결과 확인 기능 부족

#### 개선 방향

* 대기방 기능 개선
* 게임 화면 전환 기능 추가
* 게임 종료 화면 제작
* 랭크 시스템 도입
